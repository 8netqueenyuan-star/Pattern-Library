# Pattern-Library

> 识别模式、抽象结构、提炼可复用框架的总部。

## 这是什么
Pattern-Library 用来存放可复用的抽象模式、神学框架、写作模板、分析骨架与方法论。

## 为什么存在
- 识别重复出现的结构。
- 把灵感变成可复用框架。
- 为内容创作、研究和产品化提供统一语言。

## 如何使用
1. 先读 `index.md`。
2. 从 `patterns/` 找你要的抽象框架。
3. 从 `templates/` 找可直接套用的结构。
4. 每新增一个 Pattern，都补上相关 Case 链接。

## 索引规则
- 每个 Pattern 都有唯一 ID，例如 `P-001`。
- 每个 Pattern 页面底部列出 `Related Cases`。
- 每个 Pattern 页面顶部注明适用场景和失效场景。

## 双向链接
- Pattern 页面链接到对应 Case。
- Case 页面反向链接到对应 Pattern。
- Root Index 统一收口，不让链接散掉。

## 适用范围
- 神学分析。
- 内容生产。
- 双语发布。
- 一人公司知识管理。

## 维护原则
- 一个 Pattern 解决一个问题。
- 不要把案例写进 Pattern。
- 不要把抽象定义混进 Case。
